## 使用している主なフレームワーク

この gemini-cli プロジェクトでは、主に以下のフレームワークやライブラリが使用されています。

### 1. UI / CLI フレームワーク

- Ink: ターミナル上で React コンポーネントを使ってリッチな UI を構築するためのフレームワークです。
- React (v19):
  Ink のベースとして使用されています。CLI ツールですが、コードの書き方は React アプリケーションそのものです。

### 2. AI & エージェント連携

- @google/genai: Google Gemini API へアクセスするための公式 SDK です。
- @modelcontextprotocol/sdk: MCP (Model Context
  Protocol) を利用するための SDK です。これにより、AI がツールや外部データコンテキストと標準的な方法で連携できるようになっています。

### 3. 言語・開発環境

- TypeScript: プロジェクト全体が TypeScript で記述されています。
- Node.js: ランタイムとして Node.js (v20以上) が必要です。
- esbuild: 高速なビルド/バンドルツールとして使用されています。
- Vitest: テストフレームワークとして採用されています。

### 4. その他主要なライブラリ

- Zod: データのバリデーション（型チェック）に使用されています。
- Yargs: コマンドライン引数の解析に使われています。
- Fzf: あいまい検索機能の実装に使われているようです。

簡単に言うと、「React (Ink) で UI を作り、Gemini
API と MCP を組み合わせて高度な機能を提供する、TypeScript 製の CLI ツール」 という構成です。

## 配布用ファイルの作り方

1. ビルドとパッケージ作成（開発者の作業）

プロジェクトのディレクトリで以下のコマンドを実行します。これにより、配布可能な圧縮ファイル（.tgz）が生成されます。

```
bash
npm pack
```

※ このコマンドは内部で自動的にビルド（npm run bundle）も行います。

2. ファイルの配布 生成された .tgz ファイル（例:
   google-gemini-cli-0.21.0-....tgz）を、メールやチャット、共有ストレージ経由で使いたい人に渡します。

3. インストール（利用者の作業） 受け取った人は、以下のコマンドでそのファイルをインストールすることで gemini コマンドが使えるようになります。

```
bash
npm install -g ./google-gemini-cli-0.21.0-....tgz
```
